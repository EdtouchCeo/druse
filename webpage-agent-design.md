# 대륜고등학교 사용설명서 웹페이지 자동화 에이전트 — 통합 설계서

> Claude Code 구현 참조용 계획서 | 작성일: 2026-04-03 | 최종 수정: 2026-04-04

---

## 1. 작업 컨텍스트 문서

### 1.1 배경 및 목적

대륜고등학교는 업무 매뉴얼 에이전트(기존 설계서 참조)를 통해 표준화된 Word 문서를 생성하고 있다.
이 에이전트는 그 산출물과 별도로 업로드되는 학생·학부모용 안내 자료를 입력으로 받아
**교사용 / 학생·학부모용 탭으로 구분된 단일 HTML 사용설명서 웹페이지**를 자동 생성·갱신한다.

- **교사용 목적:** 반복 매뉴얼 제작 업무 경감, 도구 사용법 명확화
- **학생·학부모용 목적:** 학교 안내 정보의 정확한 전달

### 1.2 범위

| 구분 | 포함 | 제외 |
|------|------|------|
| 입력 형식 | `.docx`, `.pdf`, `.txt`, URL(`http/https`) | 이미지, HWP, 스프레드시트 |
| 입력 구분 | 교사용 설계서 / 학생·학부모용 안내 자료 | 다자 대화록, 첨부 이미지 |
| 출력 형식 | 단일 `index.html` (탭 구조) | 멀티 페이지, CMS 연동 |
| 운영 환경 | 로컬 브라우저 직접 열기 (file://) | 서버 배포, 로그인 인증 |
| 갱신 방식 | 기존 HTML 덮어쓰기 (버전 보존) | 부분 DOM 수정 |

### 1.3 입출력 정의

**입력**

| 구분 | 위치/형식 | 내용 |
|------|---------|------|
| 교사용 소스 (파일) | `/input/teacher/<파일명>` | 업무 매뉴얼 에이전트 설계서, 운영 가이드 등 |
| 학생·학부모용 소스 (파일) | `/input/student/<파일명>` | 학교생활 안내, 공지사항, 신청 절차 등 |
| 교사용 소스 (URL) | `teacher https://...` | 위와 동일한 내용의 웹페이지 링크 |
| 학생·학부모용 소스 (URL) | `student https://...` | 위와 동일한 내용의 웹페이지 링크 |

- 파일은 1개 이상, 복수 파일 누적 입력 가능
- URL 입력 시 `teacher` 또는 `student` 키워드를 URL 앞에 명시
- 실행 명령에서 대상 폴더(또는 파일 경로, URL) 지정

**출력**

| 파일 | 위치 | 설명 |
|------|------|------|
| `index.html` | `/output/web/index.html` | 최신 확정본 |
| `index_v{N}.html` | `/output/web/versions/` | 이전 버전 보존 (덮어쓰기 전 자동 백업) |

**출력 HTML 구조**

```
[헤더] 대륜고등학교 AI 에이전트 사용 설명서
[탭 네비게이션]
  ├── 탭1: 교사용 안내
  │     ├── 목적 소개 카드
  │     ├── 단계별 사용법
  │     ├── 산출물 구조
  │     ├── 설치 전제조건
  │     └── 오류 대처
  └── 탭2: 학생·학부모 안내
        ├── (자료 없을 경우) 업로드 안내 플레이스홀더
        └── (자료 있을 경우) 섹션별 안내 콘텐츠
```

### 1.4 주요 제약조건

- 출력 HTML은 **외부 CDN 없이 단독 실행 가능**해야 한다 (CSS·JS 인라인 또는 번들)
- 탭 구조는 교사 / 학생·학부모 **2개 고정**; 탭 추가는 현재 범위 외
- 입력 파일은 `teacher` / `student` **폴더로 구분**, URL 입력은 명령 앞 **키워드로 구분**; LLM이 내용으로 추측하지 않음
- 기존 `index.html`이 있을 경우 **덮어쓰기 전 자동 버전 백업**
- Human review는 Claude Code 채팅창에서 수행; 별도 인터페이스 없음
- URL 입력 시 외부 인터넷 연결 필요; 접속 불가 사이트(방화벽·로그인 필요)는 에스컬레이션

### 1.5 용어 정의

| 용어 | 정의 |
|------|------|
| 소스 파일 | `/input/teacher/` 또는 `/input/student/`에 위치한 원본 문서, 또는 URL로 지정된 웹페이지 |
| 섹션 콘텐츠 | LLM이 소스 파일을 분석하여 생성한 HTML 탭 내부 내용 블록 |
| 초안 HTML | LLM 생성 후 미검토 상태의 `index.html` |
| 확정 HTML | Human review 완료 후 `/output/web/index.html`에 저장된 최종본 |
| 버전 백업 | 덮어쓰기 전 자동 저장되는 `index_v{N}.html` |
| 플레이스홀더 | 해당 탭에 자료가 없을 때 표시되는 업로드 안내 UI 블록 |
| 누적 갱신 | 기존 탭 콘텐츠를 유지하면서 새 소스 파일 내용을 추가·병합하는 동작 |

## 2. 워크플로우 정의

### 2.1 전체 흐름도

```
[시작] 사용자: "웹페이지 만들어 /input/teacher/<파일>" 또는 "웹페이지 만들어 teacher <URL>"
  │            "/input/student/<파일>" 또는 "student <URL>"
  │
  ▼
[STEP 1] 입력 파싱
  │  - 파일 입력: 폴더 경로로 대상 탭(teacher/student) 자동 결정
  │  - URL 입력: 명령 앞 키워드(teacher/student)로 탭 결정
  │  - file-reader 스킬로 텍스트 추출 (파일: 확장자 판별 / URL: read_url.py)
  │  실패: 에스컬레이션 (파일 없음 / 미지원 형식 / URL 접속 불가)
  │
  ▼
[STEP 2] 기존 HTML 상태 확인
  │  - /output/web/index.html 존재 여부 확인
  │  - 존재 시: 버전 백업 (index_v{N}.html), 기존 탭 콘텐츠 추출
  │  - 미존재 시: 신규 생성 모드
  │
  ▼
[STEP 3] 섹션 콘텐츠 생성 ← LLM 판단
  │  - 소스 파일 내용 분석
  │  - 대상 탭(teacher/student)에 맞는 HTML 콘텐츠 블록 생성
  │  - 기존 탭 콘텐츠가 있으면 병합(누적 갱신)
  │  실패: 자동 재시도 최대 2회 → 에스컬레이션
  │
  ▼
[STEP 4] 자기 검증 ← LLM 판단
  │  - 필수 섹션 존재, 한국어 정합성, 플레이스홀더 처리 여부 확인
  │  실패: STEP 3 복귀 (재시도 횟수 차감)
  │
  ▼
[STEP 5] HTML 전체 조립 및 저장 (초안)
  │  - html-writer 스킬: 기존 탭 + 새 탭 콘텐츠 → index.html 생성
  │  - 저장 위치: /output/web/index.html (초안 표시)
  │  실패: 자동 재시도 1회 → 에스컬레이션
  │
  ▼
[STEP 6] ⏸ Human Review (대기)
  │  에이전트: "초안 위치: /output/web/index.html
  │             브라우저로 확인 후 수정 요청 또는 '승인'을 입력하세요."
  │
  ├─ "승인" → 초안을 확정본으로 표시 → [STEP 8 종료]
  │
  ▼
[STEP 7] 피드백 반영 ← LLM 판단
  │  - 수정 계획 요약 출력 후 HTML 수정
  │  - 수정된 index.html 저장
  │  실패: 자동 재시도 1회 → 에스컬레이션
  │  수정 후 STEP 6으로 복귀 (재검토 루프)
  │
  ▼
[STEP 8] 종료
  에이전트: "완료. 확정본: /output/web/index.html
             백업: /output/web/versions/index_v{N}.html"

---
[에스컬레이션 공통 동작]
  진행된 단계까지의 HTML이 있으면 /output/web/index_partial.html 저장
  오류 내용 및 직접 수정 방법 안내 출력 후 중단
```

### 2.2 LLM 판단 영역 vs 스크립트 처리 영역

| 단계 | 처리 주체 | 내용 |
|------|----------|------|
| STEP 1 파일 읽기 | 스크립트 | 확장자 기반 파서 선택 또는 URL 감지, 텍스트 추출 |
| STEP 2 HTML 파싱 | 스크립트 | 기존 index.html에서 탭 콘텐츠 블록 추출 |
| STEP 3 콘텐츠 생성 | **LLM** | 소스 분석 → 탭 섹션 HTML 블록 작성 |
| STEP 3 누적 병합 | **LLM** | 기존 콘텐츠와 신규 콘텐츠 의미 충돌 없이 병합 |
| STEP 4 자기 검증 | **LLM** | 섹션 완결성, 한국어 정합성, 플레이스홀더 처리 확인 |
| STEP 5 HTML 조립 | 스크립트 | 탭 콘텐츠 블록 + 셸(CSS·JS·헤더) → index.html 생성 |
| STEP 6 대기 | Human | 브라우저 확인 후 채팅창 피드백 |
| STEP 7 수정 반영 | **LLM** | 자연어 피드백 → HTML 수정 계획 요약 → 반영 |
| 버전 백업 | 스크립트 | 덮어쓰기 전 index_v{N}.html 복사 저장 |

### 2.3 각 단계 성공 기준 및 실패 처리

| 단계 | 성공 기준 | 검증 방법 | 실패 처리 |
|------|----------|----------|----------|
| STEP 1 | 텍스트 1자 이상 추출 (파일 또는 URL) | 스키마 검증 | 에스컬레이션 |
| STEP 2 | 기존 HTML 파싱 성공 또는 신규 플래그 설정 | 규칙 기반 | 에스컬레이션 |
| STEP 3 | 대상 탭 콘텐츠 블록 생성 완료, 필수 섹션 포함 | LLM 자기 검증 | 자동 재시도 최대 2회 → 에스컬레이션 |
| STEP 4 | 섹션 완결성 + 한국어 정합성 + 플레이스홀더 처리 | LLM 자기 검증 | STEP 3 복귀 (재시도 횟수 차감) |
| STEP 5 | index.html 파일 생성 확인, 브라우저 파싱 오류 없음 | 규칙 기반 (파일 존재 + HTML 문법) | 자동 재시도 1회 → 에스컬레이션 |
| STEP 7 | 피드백 지시 항목 전부 반영 | LLM 자기 검증 (수정 요약) | 자동 재시도 1회 → 에스컬레이션 |

> **에스컬레이션 파일:** 진행된 HTML이 있으면 `/output/web/index_partial.html`로 저장

## 3. 구현 스펙

### 3.1 폴더 구조

```
/project-root
  ├── CLAUDE.md                            # 메인 에이전트 지침 (오케스트레이터)
  ├── /.claude
  │   └── /skills
  │       ├── /file-reader                 # 기존 매뉴얼 에이전트와 공유
  │       │   ├── SKILL.md
  │       │   └── /scripts
  │       │       ├── read_docx.py
  │       │       ├── read_pdf.py
  │       │       ├── read_txt.py
  │       │       └── read_url.py          # URL 웹페이지 텍스트 추출 (내장 모듈)
  │       └── /html-writer
  │           ├── SKILL.md                 # HTML 조립 규칙, 셸 템플릿 위치
  │           ├── /scripts
  │           │   ├── build_html.py        # 탭 콘텐츠 블록 + 셸 → index.html
  │           │   ├── parse_html.py        # 기존 index.html에서 탭 블록 추출
  │           │   └── backup_html.py       # 덮어쓰기 전 버전 백업
  │           └── /references
  │               └── shell_template.html  # CSS·JS·헤더 포함 기본 셸
  ├── /input
  │   ├── /teacher                         # 교사용 소스 파일 투입
  │   └── /student                         # 학생·학부모용 소스 파일 투입
  ├── /output
  │   └── /web
  │       ├── index.html                   # 확정본 (항상 최신)
  │       └── /versions
  │           └── index_v{N}.html          # 자동 버전 백업
  └── /docs
      └── section_guide.md                 # 탭별 권장 섹션 구조 참고
```

### 3.2 에이전트 구조

**단일 에이전트** (CLAUDE.md 오케스트레이터)

선택 근거: 워크플로우가 순차적이고 탭이 2개로 고정되어 있으며,
콘텐츠 생성과 HTML 조립 책임이 명확히 분리되어 서브에이전트 없이
스킬 호출만으로 충분히 처리 가능하다.

### 3.3 CLAUDE.md 핵심 섹션 목록

1. **역할 정의** — 이 에이전트의 목적과 실행 범위
2. **실행 트리거** — `"웹페이지 만들어 [파일경로]"` 한 문장으로 STEP 1 시작
3. **입력 유형 및 탭 결정 규칙** — 파일: 폴더 경로 기반 / URL: 명령 앞 키워드(`teacher`/`student`) 기반
4. **누적 갱신 프로토콜** — 기존 index.html 존재 시 탭 추출·병합 절차
5. **워크플로우 단계별 지침** — STEP 1~8 순서대로 행동 지침
6. **Human Review 대기 프로토콜** — 안내 메시지 형식, "승인" 처리, 재검토 루프
7. **스킬 호출 규칙** — 각 스킬의 호출 시점 및 입출력 계약
8. **플레이스홀더 처리 규칙** — 해당 탭 자료 없을 때 표시할 UI 블록 기준
9. **자기 검증 체크리스트** — 필수 섹션, 한국어 정합성, 독립 실행 가능 여부
10. **에스컬레이션 프로토콜** — 재시도 소진 시 partial 저장 및 사용자 안내 메시지 형식
11. **버전 네이밍 규칙** — `index_v1.html`, `index_v2.html`... 시리얼 증가

### 3.4 스킬 목록

| 스킬명 | 역할 | 트리거 조건 | 입력 | 출력 |
|--------|------|------------|------|------|
| `file-reader` | 입력 파일 확장자 감지 또는 URL 판별 후 텍스트 추출 | STEP 1: 파일 경로 또는 URL 수신 시 | `/input/teacher/`, `/input/student/` 내 파일 또는 `http(s)://` URL | 추출된 텍스트 문자열 |
| `html-writer` (parse_html) | 기존 index.html에서 교사/학생 탭 콘텐츠 블록 분리 추출 | STEP 2: 기존 HTML 존재 시 | `/output/web/index.html` | 탭별 HTML 블록 문자열 |
| `html-writer` (build_html) | 탭 콘텐츠 블록 + 셸 템플릿 → 완성 HTML 생성 | STEP 5: 콘텐츠 검증 통과 후 | 교사 탭 블록, 학생·학부모 탭 블록, 셸 템플릿 경로 | `/output/web/index.html` |
| `html-writer` (backup_html) | 덮어쓰기 전 버전 자동 백업 | STEP 2: 기존 HTML 존재 확인 직후 | 기존 `/output/web/index.html` | `/output/web/versions/index_v{N}.html` |

### 3.5 주요 산출물 파일 형식

| 파일 | 형식 | 내용 |
|------|------|------|
| `index.html` | HTML (단일 파일) | 확정본. CSS·JS 인라인 포함, 외부 의존 없음 |
| `index_v{N}.html` | HTML | 버전 백업본 |
| `index_partial.html` | HTML | 에스컬레이션 발동 시 부분 완성본 |

**셸 템플릿 구성 요소 (`shell_template.html`)**

```
- <head>: 인라인 CSS (폰트, 색상 변수, 탭 레이아웃, 카드 컴포넌트)
- <header>: 학교명, 에이전트명, 부제
- <nav>: 탭 버튼 2개 (교사용 / 학생·학부모)
- <main>: 탭 패널 마운트 포인트 2개 (콘텐츠 블록 삽입 위치)
- <script>: 탭 전환 JS (인라인)
```

**LLM이 생성하는 탭 콘텐츠 블록 구성 (교사 탭 예시)**

```
[목적 소개 카드]          ← 2줄 요약, 경감 효과 강조
[단계별 사용법]           ← numbered step UI, 주요 명령어 코드 블록
[산출물 구조]             ← 생성 파일 목록 표
[설치 전제조건]           ← 설치 항목 그리드
[오류 대처]               ← 상황별 에이전트 동작 표
```

**LLM이 생성하는 탭 콘텐츠 블록 구성 (학생·학부모 탭)**

학생·학부모 탭은 5개 고정 카테고리로 구성된다.
각 카테고리는 정보 성격에 맞는 UI 컴포넌트를 사용하며,
자료 미입력 시 해당 카테고리에 플레이스홀더를 표시한다.

| 카테고리 | UI 컴포넌트 | 이유 |
|---------|-----------|------|
| 📜 규정·기준 | 아코디언 (항목별 펼침) | 항목이 많고 선택적으로 열람 |
| 🏫 학교생활 | 카드 그리드 (4칸) | 핵심 수치를 한눈에 비교 |
| 📚 과목 선택 | 단계 흐름 (Flow Steps) | 순서가 있는 절차 정보 |
| 💬 상담 방법 | 연락처 카드 그리드 | 상담 유형별 접근 경로 명확화 |
| 🎓 진학 정보 | 카드 그리드 + 일정 표 | 날짜·수치 중심 정보 강조 |

카테고리 간 이동은 상단 네비게이션 버튼으로 처리한다 (탭 내 탭 구조).

### 3.6 데이터 전달 방식

| 구간 | 방식 | 이유 |
|------|------|------|
| STEP 1 → STEP 3 | 프롬프트 인라인 | 추출 텍스트를 LLM이 바로 분석 |
| STEP 2 → STEP 3 | 프롬프트 인라인 | 기존 탭 블록을 병합 프롬프트에 직접 포함 |
| STEP 3 → STEP 5 | 파일 기반 (`/output/web/`) | HTML 블록이 크고 구조화되어 있음 |
| STEP 6 피드백 | 채팅창 → 프롬프트 인라인 | 피드백이 짧고 자유형식 자연어 |

## 4. 설계 결정 사항 (확정)

| 항목 | 결정 내용 |
|------|----------|
| 탭 구성 | 교사용 / 학생·학부모용 2개 고정 (학생·학부모 탭은 5개 카테고리 내부 네비게이션) |
| 탭 결정 방식 | 파일 입력: 폴더 기반 (`/input/teacher/`, `/input/student/`) / URL 입력: 명령 앞 키워드(`teacher`/`student`) — LLM 내용 추측 없음 |
| 외부 의존성 | 없음 — CSS·JS 전부 인라인 또는 번들 (file:// 직접 열기) |
| 기존 HTML 처리 | 덮어쓰기 전 자동 버전 백업 + 탭별 콘텐츠 추출·병합 |
| 버전 백업 네이밍 | 시리얼 증가 (`index_v1.html`, `index_v2.html`...) |
| 학생·학부모 자료 미입력 시 | 플레이스홀더 UI 표시 (업로드 안내 메시지) |
| 재시도 횟수 | STEP 3: 최대 2회, 나머지: 최대 1회 |
| Human review 루프 | 승인 전까지 STEP 6→7 반복 허용 (횟수 제한 없음) |
| 빠른 경로 | STEP 6에서 "승인" 입력 시 수정 없이 확정 |
| PDF 파싱 | `pymupdf` (스캔본 OCR 제외) |
| `.docx` 파싱 | `python-docx` |
| `file-reader` 스킬 | 기존 매뉴얼 에이전트와 공유 |
| 에스컬레이션 동작 | `index_partial.html` 저장 + 오류 안내 출력 + 중단 |
| 실행 환경 | 업무 담당 교직원 개인 Windows PC |

## 5. 설치 전제조건

| 구성요소 | 버전 요구사항 | 설치 명령 |
|---------|------------|---------|

> **Note:** URL 입력 기능은 추가 라이브러리 없이 Python 내장 모듈(`urllib.request`, `html.parser`, `gzip`, `zlib`)만으로 동작한다.
